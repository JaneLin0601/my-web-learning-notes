Clearfix — 解决浮动塌陷的 3 种方法

1️⃣ 什么是浮动塌陷？

当子元素使用 float 后，父容器会 失去高度， 导致后续元素布局错乱。这就是 “高度塌陷”。

⚠️ 父容器高度为 0（红色边框包裹不住子元素）

/* 问题代码：父容器没高度 */
.parent {
  border: 2px solid red;
}
.child {
  float: left;
}

2️⃣ 3 种清除浮动方法

① clearfix 推荐

最经典、最干净的方法

.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

② overflow 常用

触发 BFC，一行搞定

.parent {
  overflow: hidden;
}

⚠️ 注意：会裁剪超出的内容

③ 父浮动 不推荐

让父容器也浮动

.parent {
  float: left;
  width: 100%;
}

⚠️ 可能导致更多浮动问题

3️⃣ 标准 clearfix（直接抄）

这是 最推荐 的写法，兼容所有浏览器。

/* 标准 clearfix —— 一劳永逸 */
.clearfix::after {
  content: "";
  display: table; /* 或 block */
  clear: both;
}

/* 兼容 IE 6/7 */
.clearfix {
  *zoom: 1;
}

4️⃣ 浮动 vs Flexbox

❌ 浮动（旧方案）

  • 需要清除浮动
  • 布局复杂
  • 维护困难
  • 仅用于兼容老项目

✅ Flexbox（新方案）

  • 不需要清除浮动
  • 一行代码搞定布局
  • 易于维护
  • 新项目首选

5️⃣ 记忆口诀 🧠

“浮动要清除，父容器才不塌陷”
clearfix · overflow · 父浮动
✅ 新项目用 Flexbox ⚠️ 老项目用 clearfix ❌ 别再写浮动布局了
