📍 定位 position 核心要点

5 种定位 · 父级参照 · 层级控制

1️⃣ 核心概念

定位 = 让元素脱离文档流，相对于某个参照物进行偏移。

🔑 关键点： 子级绝对定位 需要父级有 position: relative， 否则子级会相对于 整个视口（或最近的定位祖先） 定位。

💡 你问的 “子级绝对定位了，父级要相同绝对定位的意思吗？” —— 答案是 不是。

父级只需要 position: relative（或任意非 static 定位），子级 position: absolute 就会以父级为参照物。

2️⃣ 5 种定位方式

static 默认

正常文档流，不能用 left/top 偏移
📌 所有元素默认值，无需设置

relative 参照物

相对于 自身原本位置 偏移，不脱离文档流
📍 常用于 父级参照物（配合 absolute）

absolute 子级

相对于 最近的定位祖先 偏移，脱离文档流
📌 父级必须有 position: relative 才能“锚定”

fixed 固定

相对于 浏览器视口 固定，滚动时 不动
📌 常用于 导航栏、返回顶部 按钮

sticky 粘性

相对定位 + 固定定位的混合，滚动到阈值时 固定
📌 常用于 表头、侧边栏，需要设 top: 0

3️⃣ 实战演示：绝对定位 → 父级参照

下面这个例子：父级 position: relative，子级 position: absolute; left: 50px; top: 50px;

⬇️ 父容器（relative）
absolute

left: 50px; top: 50px;

✅ 子级以 父级左上角 为起点偏移 50px

/* 父级：作为参照物 */
.parent {
  position: relative;
  width: 300px;
  height: 200px;
}

/* 子级：以父级为起点 */
.child {
  position: absolute;
  left: 50px;
  top: 50px;
}

💡 “以父级为起点，又不是以父级为终点” —— 意思是：子级 起点 是父级的左上角，但 偏移量 由 left/top 决定，可以超出父级边界。

4️⃣ 层级控制：z-index

z-index 控制定位元素的 堆叠顺序，数值越大越靠上。

.box-1 {
  position: absolute;
  z-index: 10; /* 在上层 */
}
.box-2 {
  position: absolute;
  z-index: 1; /* 在下层 */
}

⚠️ z-index 只在定位元素上生效（relative / absolute / fixed / sticky）。

📌 父级 z-index 会影响整个子级堆叠上下文。

5️⃣ 记忆口诀 🧠

“父级 relative，子级 absolute，
left / top 定偏移，z-index 控层级”
relative = 参照物 absolute = 跟着父级走 fixed = 固定在视口 sticky = 滚动粘性
