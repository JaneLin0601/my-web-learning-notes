📦 盒子模型核心要点

内容 · 内边距 · 边框 · 外边距 —— 一切皆盒子

1️⃣ 什么是盒子模型

在 HTML 中，每一个元素 都是一个“盒子”。这个盒子由四层组成：

margin（外边距）
border（边框）
padding（内边距）
content（内容）
margin（外边距）

盒子与盒子之间的距离

border（边框）

盒子的边框线

padding（内边距）

内容与边框之间的空间

content（内容）

文字、图片等实际内容

2️⃣ 核心属性速查

width / height
内容的宽高
auto | 100px | 50%
padding
内边距（4 个方向）
10px | 10px 20px | 10px 20px 30px 40px
border
边框（粗细 样式 颜色）
1px solid #333 | 2px dashed red
margin
外边距（4 个方向）
10px | 10px auto | 10px 20px 30px 40px
box-sizing
尺寸计算方式
content-box（默认）| border-box（推荐）
border-radius
圆角
8px | 50%（圆形）| 10px 20px

3️⃣ ⚠️ 最关键：box-sizing

这是盒子模型里 最容易坑人 的属性，决定了宽高怎么算。

❌ content-box（默认）

内容
width: 200px + padding 40px + border 4px = 244px

⚠️ 实际宽度 = width + padding + border

✅ border-box（推荐）

内容
width: 200px = 内容 156px + padding 40px + border 4px

✅ 实际宽度 = width（包含 padding 和 border）

/* 全局推荐：让所有盒子都用 border-box 计算 */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

💡 这就是为什么很多项目在 CSS 开头写 * { box-sizing: border-box; }，防止宽高算错。

4️⃣ 常见布局场景

A. 卡片（带内边距和边框）

.card {
  background: #161b22;
  padding: 20px 25px;
  border: 1px solid #30363d;
  border-radius: 12px;
}
卡片标题

这是卡片内容，有内边距、边框和圆角。

B. 块级元素水平居中

.center-block {
  width: 300px;
  margin: 0 auto; /* 左右自动 */
}
水平居中（margin: 0 auto）

💡 必须设 width，否则 margin: 0 auto 不生效。

C. ⚠️ 外边距合并（Collapsing Margins）

上下相邻的两个块级元素，外边距会取较大值，而不是相加。

margin-bottom: 20px
margin-top: 10px

实际间距 = 20px（取较大值）

✅ 解决方案：

  • 用 padding 代替 margin
  • 父元素加 overflow: hidden
  • 用 Flexbox（子元素没有合并问题）

5️⃣ 记忆口诀 🧠

从外到内

margin → border → padding → content

速记口诀

“边（边框） 外（外边距） 内（内边距） 要分清，
盒子模型记心中。”
