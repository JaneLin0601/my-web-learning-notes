🔗 关系选择器核心要点

后代 · 子代 · 相邻兄弟 · 通用兄弟 —— 4 种关系一网打尽

1️⃣ 什么是关系选择器

关系选择器通过 元素之间的层级关系 来选中元素，而不需要给每个元素单独加类名。

✅ 优点

减少类名，HTML 更干净，样式更易维护

⚠️ 注意

选择器太深（4层以上）会影响性能，尽量控制在 3 层以内

2️⃣ 4 种关系选择器速查

后代选择器 ul li
选择 所有 后代（不管嵌套多深）
ul li → 所有 ul 里的 li
子代选择器 ul > li
只选择 直接子代（一层）
ul > li → 只有 ul 直接子级的 li
相邻兄弟选择器 h1 + p
选择 紧挨着 的下一个兄弟
h1 + p → h1 后面的第一个 p
通用兄弟选择器 h1 ~ p
选择 所有 后面的兄弟
h1 ~ p → h1 后面的所有 p

3️⃣ 后代选择器 ul li

ul li 选择 所有在 ul 内部的 li，不管嵌套了几层。

/* 所有 ul 里的 li 都变成红色 */
ul li {
  color: #ff6b6b;
}

👇 下面这个 ul 里有嵌套的 ol，里面的 li 也会被选中：

  • 第一层 li（红色）
  • 第一层 li
    1. 第二层 li（也变成红色 ✅）
    2. 第二层 li（也变成红色 ✅）
  • 第一层 li（红色）

💡 想只选第一层，用 子代选择器 ul > li（见下一节）

4️⃣ 子代选择器 ul > li

ul > li 只选择 ul 的直接子级 li，不选嵌套在里面的。

/* 只选 ul 直接子级的 li，变蓝色 */
ul > li {
  color: #64c8ff;
}

👇 只有 第一层 li 变蓝，嵌套里的不变：

  • 第一层 li（蓝色 ✅）
  • 第一层 li（蓝色 ✅）
    1. 第二层 li（不变 ❌）
    2. 第二层 li（不变 ❌）
  • 第一层 li（蓝色 ✅）

ul li → 选所有后代（深）

ul > li → 只选直接子代（浅）

5️⃣ 相邻兄弟选择器 h1 + p

h1 + p 选择 紧挨着 h1 后面的第一个 p。

/* h1 后面的第一个 p 变橙色 */
h1 + p {
  color: #d29922;
}

标题（h4）

第一个 p → 被选中 ✅

第二个 p → 没被选中 ❌

第三个 p → 没被选中 ❌

6️⃣ 通用兄弟选择器 h1 ~ p

h1 ~ p 选择 h1 后面所有的 p。

/* h1 后面的所有 p 变紫色 */
h1 ~ p {
  color: #8b5cf6;
}

标题（h4）

第一个 p → 被选中 ✅

第二个 p → 被选中 ✅

第三个 p → 被选中 ✅

7️⃣ 综合对比

选择器 名称 选择范围 示例
ul li 后代选择器 所有后代（不限层级） ul 里所有的 li
ul > li 子代选择器 只选直接子代 ul 直接子级的 li
h1 + p 相邻兄弟选择器 紧挨着的下一个 h1 后面的第一个 p
h1 ~ p 通用兄弟选择器 后面所有的兄弟 h1 后面的所有 p

8️⃣ 实战场景

📌 导航菜单（子代选择器）

nav > ul > li {
  display: inline-block;
}

只选导航第一级菜单，不选下拉子菜单

📌 文章样式（后代选择器）

.article p {
  line-height: 1.8;
}

文章容器里所有段落统一行高

📌 标题后间距（相邻兄弟）

h2 + p {
  margin-top: 0;
}

标题后面的第一个段落去掉上边距

📌 表单项间距（通用兄弟）

input + input {
  margin-top: 10px;
}

每个 input 后面的 input 加间距
